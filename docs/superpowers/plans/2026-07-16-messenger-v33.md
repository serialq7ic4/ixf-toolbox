# Messenger v3.3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the first stable Go-owned Messenger foundation release.

**Architecture:** Add `ixf messenger ...` to the CLI and isolate platform/profile logic in `internal/messenger`. The first release performs secret-safe diagnostics and dry-run planning only; real browser read/send commands remain gated for later releases.

**Tech Stack:** Go 1.24, standard library filesystem/process APIs, existing embedded skill setup.

## Global Constraints

- No Python runtime or Python scripts may be reintroduced.
- Messenger must not open or mutate the live LarkShell profile directly.
- Real message sending must not be exposed in `v3.3.0`.
- Diagnostics must not leak cookie values, CSRF tokens, message text, private IDs, or raw profile content.
- `go test ./...` and `go vet ./...` are the verification entrypoints.
